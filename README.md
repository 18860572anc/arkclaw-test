## 🎯 使用指南

### QA测试工程师
1. 阅读第0-1章了解测试策略和范围
2. 按第8.4节的执行建议顺序进行测试：
   - 先跑通E2E主流程（第2章）
   - 再按角色分域测试（第3章）
   - 然后执行权限隔离测试（第4章）
   - 接着进行UI规范检查（第5章）
   - 最后执行异常流程测试（第6章）
3. 记录测试结果和发现的缺陷
4. 对照第8章验收标准评估发布 readiness

- 使用附录A的PRD需求追踪矩阵验证需求覆盖度
- 对照每个测试用例的"关联PRD"字段确认功能符合PRD定义

### 前端开发
- 参考第5章UI规范检查点进行自测
- 关注第6章异常流程确保错误处理完善
- 使用第7章Mock数据模板准备测试环境

## 🆕 v1.1 重要更新说明

### 新增测试覆盖页面（部分示例）

**租户侧增强功能**：
- 技能中心（技能广场、企业精选）
- 应用中心（Agent/MCP注册与管理）
- 知识中心（数据源连接配置）
- 合同管理（合同列表与详情）
- 媒体资讯（资讯阅读）
- 首次开通向导（空间初始化）

**销售侧增强功能**：
- 代客使用券（优惠券抵扣下单）
- 代客支付（代为完成支付流程）

**官方管理员侧增强功能**：
- 发票管理（申请审核、开票状态）
- 外部代理管理（代理列表、余额管理）
- 代理订单总览（余额看板、采购ledger）
- 代理网站管理（镜像站配置）

**运维侧增强功能**：
- IM绑定管理（飞书/企微/钉钉配置）
- 客户诊断与支持（深度诊断、工具执行）
- 健康巡检（规则配置、自动执行）
- 客户运维档案（全景查看、历史追溯）

### 未实现功能标注

以下PRD定义的功能前端尚未完全实现，标记为"后置范围"：
- 角色权限矩阵（`/admin/system/roles`）
- 字典管理（`/admin/system/dictionaries`）
- 系统参数（`/admin/system/params`）
- 全局审计日志（`/admin/system/audit`）- 建议Beta1.1实现
- 系统监控与告警（`/admin/monitoring`）
- 资讯管理后台（组件已实现但未启用）
