## 一、产品背景与愿景

### 1.1 背景

云脑智联（以下简称"我司"）与火山引擎签署 ArkClaw 企业版转售代理协议。ArkClaw 是火山引擎推出的企业级 AI 智能体云服务，具备 Claw 实例管理、技能中心、知识中心、席位管理、安全管控、可观测等完整能力。

但火山原生控制台存在**一个关键限制**：一个火山账号只能管理一个企业空间。作为代理商，我们面向**多个下游企业客户**，必须解决：

1. **不让客户登录火山账号**：不暴露底层代理关系、不让客户跨过我们直连火山；
2. **让每个客户拥有独立的、品牌化的管理空间**；
3. **让我司官方能统一管理所有下游企业**：看全局、调策略、做支持；
4. **支撑销售团队的一级分销体系**：客户归属、佣金计算、分享链接；
5. **支撑客户自助充值 → 我司代收 → 同步火山额度**的闭环。

### 1.2 愿景

打造一个**"和 ArkClaw 长得一样，但是我们自己域名、我们自己品牌、我们自己管多个客户"**的 SaaS 管理平台，让：

- **终端企业客户**：得到和 ArkClaw 企业版一致的使用体验，但只看到自己企业的数据，感知到的是"云脑智联 × 火山引擎"联合品牌；
- **我司销售**：有专属工作台，自动归因客户、实时预估佣金；
- **我司交付运维**：有独立工具对接飞书/企微/钉钉/微信绑定、做网络配置和客户支持；
- **我司官方管理员**：拥有上帝视角，管所有企业、所有订单、所有销售、所有分成。

### 1.3 设计原则

1. **UI 对齐火山**：组件、栅格、色彩、术语尽量沿用火山 ArkClaw 视觉，降低客户切换成本、强化"和火山是一家"的联合品牌感。建议 UI 组件库优先选 **Arco Design**（火山出品）。
2. **能力透传 ≥ 90%**：凡是火山 OpenAPI 已覆盖的能力，直接透传到客户侧界面，不做二次加工；没有 API 的暂不提供。
3. **数据严格多租户隔离**：每个客户企业 = 一个独立 tenant，数据库行级隔离 + 应用层 RBAC 双保险。
4. **代理商不碰业务数据**：Claw 会话内容、文件、知识库等业务数据归客户所有，我司仅保留元数据、用量、审计。
5. **销售归因一次且终身**：一个客户被一个销售首次绑定后，12 个月内产生的付款按固定 5% 分成，不支持多级。

### 1.4 已确认的业务规则

| 编号 | 规则 | 决策 |
|---|---|---|
| D1 | 佣金分成周期 | **客户首次付款后 12 个月** |
| D2 | 佣金比例 | **固定 5%**（无阶梯） |
| D3 | 客户去重主键 | **统一社会信用代码**（企业唯一标识） |
| D4 | 充值链路 | **我司代收**（客户付我司、我司按需在火山侧扣额度） |
| D5 | 支付渠道 | **微信 + 支付宝 + 对公转账**全要 |
| D6 | 咨询智能体 | **云脑智联自建客服智能体** |
| D7 | 分销入口 | **分享链接 + 邀请码**两者都要 |
| D8 | 客户经理分配 | **首版手动**，预留自动分配扩展点 |
| D9 | 媒体资讯发布主体 | **我司发布**（"官方资讯"一栏；企业客户可阅读，不对客户开放发布） |

## 二、目标与非目标

### 2.1 本期目标（In Scope）

1. **多租户管理平台**：客户企业的全生命周期管理，对齐 ArkClaw 企业版管理台的全部可 API 化功能。
2. **客户自助充值 + 账单**：微信 / 支付宝 / 对公三渠道，自动到账，实时更新火山侧额度。
3. **官方管理员总控台**：企业列表、订单、用量、统计、销售管理、佣金规则、资讯发布。
4. **销售工作台**：客户池（含去重）、分享链接、邀请码、销售仪表盘、佣金实时预估。
5. **交付运维工作台**：飞书/企微/钉钉/微信绑定、网络配置、工单支持。
6. **企业套餐咨询表单 + 留资分发**：客户填写需求，自动分配客户经理（首版手动）。
7. **媒体资讯发布（我司侧）**：我司发布行业动态、产品更新，客户侧阅读，可嵌入咨询智能体解答。
8. **品牌化**：云脑智联 × 火山引擎双 logo、登录页、关于页、邮件模板统一。
9. **完整权限体系**：5 类角色 × 按钮级 RBAC。
10. **审计与合规**：所有对租户的敏感操作留痕。

### 2.2 明确非目标（Out of Scope）

1. ❌ **桌面客户端**（Electron 方案已单独立项，本期不做）。
2. ❌ **多级分销 / 代理商下还有代理商**：只做一级，避免传销风险。
3. ❌ **客户端 MCP Host / 本地终端能力**：属于桌面端范畴。
4. ❌ **火山没提供 API 的能力**：例如极深度的运维操作、火山底层基础设施配置；短期不做二次开发，界面上以"请联系客户经理"承接。
5. ❌ **客户自行发布资讯 / 企业新闻管理**：媒体资讯仅我司发布，不做客户 CMS。
6. ❌ **自建 LLM / 自建 Agent 框架**：所有 AI 能力走 ArkClaw。
7. ❌ **多地域 / 海外部署**：首版仅国内，华北/华东节点由 ArkClaw 本身提供。
8. ❌ **移动端原生 App**：首版仅 Web，响应式布局兼容移动浏览器即可。
9. ❌ **开放平台 / 让客户调我们的 API**：客户只通过界面使用；客户自己的开发集成直接走火山 API。
10. ❌ **发票系统的完整开票流程**：首版留资、人工开票；不接税控。

### 2.3 非功能性需求（NFR）

| 维度 | 指标 |
|---|---|
| 可用性 | 核心页面 99.5% 月度可用性 |
| 性能 | 登录后首屏 ≤ 2s（P95）；列表页查询 ≤ 1s（P95） |
| 并发 | 支持 100 个企业客户 × 每企业 50 员工同时在线 |
| 数据隔离 | 任意接口必须校验 `tenant_id`，越权访问必拦截 |
| 安全 | HTTPS 全站、密码 bcrypt、敏感信息脱敏、XSS/CSRF/SQLi 防护 |
| 审计 | 官方管理员、销售、交付、企业管理员的所有写操作全量审计，保留 ≥ 12 个月 |
| 合规 | 符合《个保法》《数安法》；客户数据存储在境内 |
| 浏览器 | Chrome / Edge / Safari 最近 2 版；Firefox 最新版；IE 不支持 |
| 响应式 | 桌面优先；平板可用；手机仅支持登录、账单、咨询表单 |
| 国际化 | 首版仅中文简体；预留 i18n 框架 |

## 三、目标用户与角色

### 3.1 角色定义

| 代号 | 角色 | 说明 |
|---|---|---|
| **R1** | **我司官方管理员**（Platform Admin） | 云脑智联员工，全平台最高权限 |
| **R2** | **我司交付运维**（Delivery Ops） | 云脑智联员工，负责客户开通后的绑定、网络、技术支持 |
| **R3** | **我司销售**（Sales） | 云脑智联员工，负责拓展和维护客户，拿佣金 |
| **R4** | **客户企业管理员**（Tenant Admin） | 下游企业的管理员，管本企业内部 |
| **R5** | **客户企业成员**（Tenant Member） | 下游企业的普通员工，使用 Claw 做工作 |

### 3.2 核心用户故事

**US-01**｜作为 **R1（官方管理员）**，我希望在一个控制台看到平台上所有企业客户的运营状况（Claw 数、活跃用户、Token 消耗、账单），以便做经营决策。

**US-02**｜作为 **R1**，我希望能为任意客户企业开通 / 暂停 / 调整席位，以便做销售支持和运营干预。

**US-03**｜作为 **R1**，我希望能配置佣金规则、套餐价格、发布官方资讯，而不需要开发介入。

**US-04**｜作为 **R2（交付运维）**，我希望有专门的工具对接飞书/企微绑定、配置网络出入口，而不让客户接触这些复杂配置。

**US-05**｜作为 **R3（销售）**，我希望生成我的专属分享链接，客户通过这个链接注册后自动归属到我名下。

**US-06**｜作为 **R3**，我希望实时看到"我的客户数、本月回款、预估佣金"，以便判断业绩和优先级。

**US-07**｜作为 **R3**，新增客户时如果统一社会信用代码已被其他销售录入，系统应拦截并提示"已被他人录入"。

**US-08**｜作为 **R4（客户企业管理员）**，我希望自助充值（月/季/年，微信/支付宝/对公），无需联系销售即可续费。

**US-09**｜作为 **R4**，我希望管理本企业的员工、部门、席位分配、Claw 实例、安全策略，像在 ArkClaw 原生控制台一样。

**US-10**｜作为 **R4**，我希望看到企业用量趋势、Token 消耗排行、审计日志，满足内部合规要求。

**US-11**｜作为 **R4**，我希望当我需要升级套餐或咨询高级功能时，能填一个表单，有客户经理联系我。

**US-12**｜作为 **R5（企业成员）**，我希望通过企业微信/飞书/钉钉/微信直接唤起 Claw 工作，不需要学额外的工具。

**US-13**｜作为任意角色，当我看到平台页面时，感受到的是"云脑智联 × 火山引擎"联合品牌，而不是纯火山。

## 四、核心业务流程

### 4.1 客户获取与开通

```
销售生成分享链接 / 邀请码
  │
  ▼
客户通过链接访问落地页，填企业信息注册
  │
  ▼
销售 ID 自动写入客户记录（归属绑定）
  │
  ▼
统一社会信用代码去重校验
  │  ├─ 重复 → 提示 "该客户已由其他销售录入"
  │  └─ 通过 → 建立 pending 客户
  ▼
销售 / 客户选择套餐 → 下单 → 付款（微信/支付宝/对公）
  │
  ▼
支付成功回调
  │
  ▼
调用火山 OpenAPI：创建空间 / 分配席位 / 初始化
  │
  ▼
本系统 tenant_id ↔ 火山 space_id 建立映射
  │
  ▼
客户邮件 + 短信通知：登录链接、初始管理员密码
  │
  ▼
客户管理员首次登录 → 修改密码 → 配置企业信息
  │
  ▼
触发佣金流水（12 个月归属期开始）
```

### 4.2 客户充值续费

```
客户管理员 → 自助充值 → 选周期（月/季/年）+ 渠道 → 支付
  │
  ▼
支付成功 → 本地订单状态变更 → 调火山 API 同步额度
  │
  ▼
账单页更新 + 销售佣金流水追加（若仍在归属期）
```

### 4.3 飞书 / 企微 / 钉钉 / 微信绑定

该流程由 **R2 交付运维** 执行，**不对客户企业管理员开放**（因为涉及 AgentID / Secret / 回调 URL 等技术参数）：

```
客户下单成功 → 工单进入交付队列
  │
  ▼
R2 在"交付工作台"选择该客户
  │
  ▼
按引导配置 企微/飞书/钉钉/微信 的回调地址、AgentID、Secret
  │
  ▼
测试联通 → 保存配置 → 通知客户管理员"员工可登录"
```

### 4.4 销售佣金实时计算

```
任意客户订单支付成功
  │
  ▼
查询该客户的归属销售 + 首次付款时间
  │
  ▼
若当前时间在"首次付款 + 12 个月"内 → 产生佣金流水
  │     金额 = 订单金额 × 5%
  │
  ▼
销售仪表盘实时刷新
```

## 六、模块 A：客户侧详细规格

> 设计基准：完全对齐 `/0-assets/ArkClaw` 截图的 ArkClaw 企业版控制台，在此基础上替换品牌、裁剪无 API 能力、增加自助充值/咨询/资讯三大增值板块。

### A0 空间概览

**对应截图**：`0-空间概览@1x.png`、`0-空间概览 - 模型配置编辑@1x.png`

   - 允许员工使用复制粘贴操作（已启用 / 未启用）

   - 允许员工使用 Claw 同时在线
   - 允许员工使用预订
   - 【更多配置】跳转 A6.2 员工工作台配置
6. **网络配置入口卡片**：显示配置状态 + "去开通"按钮（对接 C3 由交付运维操作；客户端只展示状态不让改）

- 四档席位每档可设置：默认模型、默认模型套餐、可选模型列表、Token 限流
- 数据源：火山 ArkClaw API 的模型列表
- 权限：仅 **R4 客户企业管理员** 可编辑

#### A0.3 成功标准
- ✅ 所有卡片数据来自火山 API，T+0 展示
- ✅ 模型配置保存后 ≤ 5s 在火山侧生效
- ✅ 响应式布局：1440px 宽屏最佳，1280px 可用

#### A0.4 非目标
- ❌ 不提供 A0 页面的 CSV 导出（留到 A5 可观测模块做）

### C3 网络配置
- 同 A4.3，但为 R2 的可编辑版本
- 自定义域名证书上传、公网跨境加速、私网出口 VPC 配置

### C4 客户诊断与支持
- 诊断工具：拉取客户最近错误日志、网络连通性检测
- 远程协助：授权后以客户管理员身份登录（审计强制）

## 九、模块 D：销售后台详细规格

### D1 销售仪表盘
- **卡片**：我的客户数、本月新客数、本月回款总额（GMV）、本月预估佣金、12 个月内归属期客户数
- **图表**：近 30 天回款趋势、客户消耗 Top 10
- **待办**：新分配的咨询留资、超时未跟进客户

### D2 我的客户池
- 增删改查（新增时触发 D3 去重校验）
- 字段：客户名、统一社会信用代码、联系人、电话、行业、归属时间、最近跟进、状态（潜在 / 已签约 / 流失）、充值额、消费额、客户经理
- 支持搜索、筛选、导出
- **客户详情**：基本信息、订单历史、消耗趋势、跟进记录、佣金流水
- **客户经理**：首版销售手动填写备注字段，不绑定系统角色

### D3 分享链接 / 邀请码
- **分享链接生成**：后端短链 + 销售 ID 加密参数；展示二维码 + 复制按钮
- **邀请码**：6 位字母数字，销售可生成多个，每个可限次数 / 限时
- **落地页**：带销售归因参数的注册页，注册成功后自动建立 sales_customer 绑定
- **防刷**：同 IP 同一小时内注册限 5 次；同手机号/邮箱不可二次归属

### D4 佣金明细
- 列表：订单号、客户、订单金额、佣金金额、计算依据（5% × 金额 × 在期）、发放状态、时间
- 导出 CSV

| 计费 | 查询额度 / 扣减 | A7 |

### 13.3 数据同步策略
- **写操作**：同步调用火山 API，成功才提交本地事务；失败回滚并提示用户
- **读操作**：优先本地缓存（分钟级 TTL），列表页按需穿透
- **增量**：核心实体（Claw、员工、席位）每 5 分钟定时对账

### 13.4 故障降级
- 火山 API 不可用 → 只读页面可用（本地缓存）、写操作返回友好错误 + 排入重试队列
- 账单/充值链路 = 强一致，不允许本地先扣费后同步火山

## 十七、关键风险

| 风险 | 影响 | 应对 |
|---|---|---|
| 火山 API 不足以覆盖本 PRD 定义的功能 | 某些页面无法上线 | 前置 API 清单对齐；无 API 能力以"请联系客户经理"过渡 |
| 火山多租户能力 roadmap 与我方重合 | 重复开发 | 每月与火山 BD 同步 roadmap |
| 火山 UI 改版影响我方界面一致性 | 体验割裂 | 建立 ArkClaw 控制台每周人工巡检 |
| 代理商代收款 → 同步火山额度的结算模型 | 资金流 / 税务风险 | 财务 + 法务前置评审；严格对账 |
| 客户数据归属纠纷 | 法律风险 | EULA / DPA 明确：业务数据归客户、元数据归我司 |
| 销售归因争议（客户跨销售签约） | 内部冲突 | 以统一社会信用代码 + 首次绑定时间为唯一依据，系统强执行 |
| 佣金规则变更（未来可能调阶梯） | 规则迁移复杂 | `commission_rule` 表预留，按订单时点锁定规则版本 |
