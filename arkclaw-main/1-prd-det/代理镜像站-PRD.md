### 1.3 术语对齐（详见 § 17.1）

- **代理（Agent）**：拥有镜像站的合作伙伴，**非我方员工**
- **代理管理员（Agent Admin）**：代理公司的"超级管理员"，拥有镜像站全部能力
- **代理销售（Agent Sales）**：代理公司的销售员，拥有销售功能（同我方销售 PRD）
- **代理财务（Agent Finance）**：代理公司的财务人员，拥有财务结算功能（同我方财务 PRD 精简版）
- **进货（Procurement）**：代理向平台全款购入产品额度
- **余额（Balance）**：代理已进货金额 - 已扣减客户开通成本 - 已提现的剩余数字（本期仅"数字运算"，非真实钱包）
- **客户成本扣减**：当代理体系下客户成功开通 → 按"代理进货价"从代理余额扣减
- **代理售卖价 vs 代理进货价 vs 平台基准价**：
  - **平台基准价**：平台 § B6 设的官方价
  - **代理进货价**：基准价 × 代理协议比例（如 70%）—— 不一定要 30% 佣金率，进货价可独立约定
  - **代理售卖价**：代理在镜像站设的卖给最终客户的价（可低可高）
  - 代理利润 = 售卖价 − 进货价 + 平台返回的代理佣金（如 30%）
- **客户透传**：代理体系下注册的客户数据全部进入平台同一个 tenant 表，归属代理但接受平台统一交付

### 1.4 设计原则

1. **代理看不到其他代理 / 直营数据**：任何接口、列表、统计强制 `WHERE agent_id = current_user.agent_id`
2. **品牌可视化定制 + 后端共享**：代理感知不到这是"同一个平台"
3. **客户体验一致**：代理客户的工作台体验和直营客户完全一样（火山 ArkClaw 透传体验）
4. **代理对客户透明**：客户看到的是代理的品牌，但底层服务和交付都是我方
5. **本期数字化运营 + 工单化结算**：余额、佣金、提现都是"数字 + 人工工单"，本期不接真实钱包 / 支付链路
6. **三角色权限严格隔离**：代理管理员 ≠ 代理销售 ≠ 代理财务，任意越权拒绝
7. **不能再发展下级代理**：镜像站内**没有**"添加代理"入口

## 二、模块定位、目标与设计原则

### 2.1 模块定位

代理镜像站是平台**给代理使用的工作台子集**，承载：

```
代理首次登录
  ↓
配置镜像站品牌（logo / 标题 / 主题色 / 自定义域名）
  ↓
配置套餐售卖价（基于平台基准价）
  ↓
向平台进货（充值进余额账户）
  ↓
招销售（在镜像站内开销售账号）
  ↓
销售对外卖（分享链接 / 邀请码 / 手动录入客户）
  ↓
客户付款 → 平台开通 → 自动扣代理余额（按进货价）
  ↓
代理财务对账 → 提现申请 → 平台财务线下打款
```

### 2.2 三角色范围（In Scope）

| 角色 | 主要工作台 | 关键差异于平台版本 |
|---|---|---|
| **代理管理员**（Agent Admin） | M1-M9 + M10 / M11 只读 | 加了 M5 进货 + 余额、M7 提现、M8 品牌、M9 价格 |
| **代理销售**（Agent Sales） | M11 销售工作台 | 与平台销售 PRD 完全一致，但 agent_id 限定 |
| **代理财务**（Agent Finance） | M10 财务工作台 + M7 提现处理 | 与平台财务 PRD 精简一致；新增提现申请入口 |

### 2.3 非目标（Out of Scope，详见 § 12）

- ❌ 跨代理数据查看（不可能看到别家代理）
- ❌ 平台层面的运营 / 配置（属超管 PRD）
- ❌ 真实钱包 / 第三方支付集成（本期数字 + 工单；V2）
- ❌ 代金券系统（5/15 会议明确不做，本期）
- ❌ 区域防串货 / 企查查接入（V2）
- ❌ 客户业务数据操作（Claw / 技能 / 用量 → 客户企业管理员侧）
- ❌ 交付运维（IM 绑定 / 网络配置 → 平台 R2，代理不接触）
- ❌ 招下级代理（防传销）

## 五、核心业务流程

### 5.1 代理首次登录引导

```
代理管理员收到平台开通通知（短信 / 邮件）
        │
        ▼
登录镜像站后台（域名 = agent-x.<我方根域> 或自定义）
        │
        ▼
首次登录强制走 4 步向导：
   Step 1：上传 logo / 改标题 / 选主题色 / 设默认欢迎页
   Step 2：基于平台基准价，设我家套餐售卖价
   Step 3：进货充值（默认带"首次代理费已充入 X 元"）
   Step 4：邀请第一个销售（输入手机 / 邮箱 → 系统发邀请）
        │
        ▼
完成引导 → 进入 M1 工作台首页
        │
        ▼
工作台显眼位置显示"已完成 X / 4 步"，可重新进入未完成步骤
```

### 5.2 进货充值流程

```
代理管理员在 M5 点"进货"
        │
        ▼
选择充值金额（自由输入 ≥ 10000 元）+ 支付方式（对公 / 微信 / 支付宝）
        │
        ▼
对公：展示我方银行信息 + 唯一备注号 + 上传凭证
微信 / 支付宝：跳支付，回调
        │
        ▼
平台官方管理员（超管 § 6.12.5）审核对公 / 自动回调
        │
        ▼
agent_order.status=paid + agent_balance += amount
        │
        ▼
代理收到通知"已到账 X 元，当前余额 Y 元"
```

### 5.3 客户开通触发余额扣减

```
代理体系下客户付款（任意路径：销售卖 / 客户自助）
   ├─ 客户付的是"代理售卖价"（进入代理的销售现金流）
   ▼
平台开通任务推进 → 企业席位池开通可用
   ├─ 同时触发：
   │    ① 平台从代理余额扣减"进货价"（代理向平台进的成本）
   │    ② 平台向代理记一笔 agent_commission（按代理协议费率，如 30%）
   ▼
代理 agent_balance -= 进货价
代理 agent_commission += 客户付款 × commission_rate
   ▼
两者都在 M6 佣金管理可见
```

> 关键：代理"销售现金流"（客户付的）和"代理佣金"（平台返的）是两笔不同的钱。
> - 销售现金流：客户付给代理（理论上代理可全部提走）
> - 代理佣金：平台另外补贴给代理

### 5.4 代理销售卖货流程（同平台销售 PRD）

```
代理销售在镜像站后台 M11
   ↓
生成分享链接 / 邀请码（带 agent_id + sales_id 双重归因）
   ↓
客户通过链接落地到镜像站注册页（看到的是代理品牌）
   ↓
客户注册 + 付款（付的是代理售卖价）
   ↓
平台开通客户 → 5.3 触发余额扣减 + 代理佣金 + 销售佣金
```

### 5.5 销售内部分佣（代理自定义）

```
代理管理员在 M6 配置"内部销售佣金率"
   · 默认：客户付款 × 5%（与平台直营销售一致）
   · 可改：代理自由设
   ↓
代理体系下客户付款触发：
   ① 平台给代理 30% 代理佣金
   ② 代理内部给销售 X%（代理自留 30% - X%）
   ↓
两者都通过 agent_commission_distribution 表记账
```

### 5.6 提现流程

```
代理管理员 / 代理财务在 M7 发起提现申请
   · 选择提现类型：销售现金流 / 代理佣金
   · 输入提现金额（≤ 当前可提余额）
   · 选择收款账户（代理预存账户）
   ↓
创建 agent_withdrawal 工单 (status=pending)
   ↓
工单透传到平台官方管理员（超管 § 6.12.7）
   ↓
平台财务线下打款 + 回写"已打款 + 银行流水号"
   ↓
代理收到通知 + agent_withdrawal.status=paid
```

> 本期机制：余额是"数字运算"，提现申请走人工。V2 上真实钱包后会自动化。

### 5.7 代理被冻结 / 关闭

```
平台冻结代理（超管 § 6.12.3 高危操作）
   ↓
镜像站访问被拦截（403 + 提示"请联系平台客服"）
名下所有销售账号同步冻结
   ↓
（关闭场景）名下客户由平台接管为直营 + 代理体系下未发佣金转入"待清算"
```

### 6.1 M1 代理工作台首页

#### 路径
`/mirror/dashboard`

#### 6.1.1 设计原则

沿用超管 PRD § 6.1 v1.2 的四区分层（告警 / 待办 / 经营 / 健康），但：
- 数据范围全部限定到 `agent_id = current_user.agent_id`
- 待办内容不同（代理关注：余额预警、客户开通进度、提现状态、未处理销售工单）
- 经营指标不同（关注：本月销售额、本月余额扣减、本月代理佣金、本月销售人数）

#### 6.1.2 页面结构

```
┌────────────────────────────────────────────────────────────────────┐
│  ⚠️ 异常告警条（仅有告警时出现）                                     │
│  示例：⚠️ 余额仅剩 ¥12,300，请尽快补货  │  本月有 1 笔提现待平台处理  │
└────────────────────────────────────────────────────────────────────┘

┌──────────── 第 1 区 · 今日待办 ────────────────────────────────────┐
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐                      │
│  │ 🔴 余额│ │ 🟡 待开│ │ 🟠 提现│ │ 🔵 销售│                      │

-- 代理余额账户
CREATE TABLE agent_balance (
  agent_id        BIGINT PRIMARY KEY REFERENCES agent(id),
  current         DECIMAL(14,2) NOT NULL DEFAULT 0,    -- 当前余额
  total_topup     DECIMAL(14,2) NOT NULL DEFAULT 0,    -- 累计进货
  total_deduct    DECIMAL(14,2) NOT NULL DEFAULT 0,    -- 累计扣减
  total_withdraw  DECIMAL(14,2) NOT NULL DEFAULT 0,    -- 累计提现
  warn_threshold  DECIMAL(12,2) NOT NULL DEFAULT 20000,-- 黄色预警
  alert_threshold DECIMAL(12,2) NOT NULL DEFAULT 5000, -- 红色告警

-- 余额流水
CREATE TABLE agent_balance_log (

## 九、业务规则细则（关键边界，需 sign-off）

### M1 数据隔离强制

- 后端中间件强制注入 `WHERE agent_id = current_user.agent_id`
- 任意接口绕过该过滤 → 401 + 高优安全告警 + 自动通知平台 SU-
- 单元测试覆盖：随机访问其他代理的资源 ID，必须返回 401 / 404

### M2 余额扣减原子性

- 客户付款触发开通 → 在**同一事务**中：① 平台开通；② 代理余额扣减；③ 写 agent_balance_log
- 余额不足：客户订单 status=blocked_insufficient_balance，**不自动退款**，进入"等代理补货"队列，通知代理 + 客户经理
- 代理补货后：自动重试开通，成功后扣减

### M3 进货价的计算口径

- 平台基准价 × 代理协议进货比例（默认 70%，可协议调）
- 老订单按下单时点的进货比例锁定
- 代理升级协议（如 70% → 65%）不溯及既往

### M4 售卖价下限

- 售卖价 < 进货价 → 系统拒绝
- 进货价 ≤ 售卖价 < 基准价 × 50% → 弹窗警告 + 二次确认
- 售卖价 ≥ 基准价 × 50% → 直接保存

### M5 内部销售佣金率上限

- 代理设给销售的佣金率 ≤ 代理利润空间（避免代理亏本）
- 系统提示但不强制：默认 = 代理佣金率 / 3（如代理拿 30%，建议销售 10%）

### M6 销售归属

- 客户通过分享链接归属销售 → 同步归属代理（agent_id = sales.agent_id）
- 客户手动录入 → 销售选属 → 自动带 agent_id
- USCC 全平台唯一：同 USCC 不能被两个代理的销售同时录入

### M7 提现校验

- 申请金额 > 可提现 → 拒绝
- 同时只能 1 个 pending 申请
- 代理财务审核内部流程；终审走平台财务（线下打款）

### M8 镜像站冻结

- 代理被平台冻结 → 镜像站访问拦截（403）+ 名下销售 / 财务账号同步冻结
- 已签约客户不受影响（继续可用）
- 解冻后镜像站立即恢复访问

### M9 客户透传与归属不可篡改

- 客户的 tenant.agent_id 在注册时落库，**永久不可改**
- 客户跨代理"转籍" → 必须平台 SU- 介入（超管 § 5.5 流程）
- 防代理之间私下"抢客户"

### M10 价格历史

- 代理改套餐售卖价 → 老订单按下单时点价格执行
- 价格变更 + 24h 内不允许再调（防误操作）

### M11 收款账户安全

- 银行账号 KMS 加密存储
- 修改账户必须代理管理员二次确认 + MFA
- 修改后 24 小时冷静期不允许立即发起提现

### M12 一级代理硬约束

- 镜像站后台**没有**"开代理"入口
- 代理无法新增 role=agent_admin 用户
- 数据库 + 应用层双保险

```
（平台开通）→ active ──[平台冻结]──→ frozen ──[平台解冻]──→ active
                                                                    
              active ──[平台关闭]──→ closed（不可恢复，需新签约）

pending ─[微信/支付宝回调]─→ paid
   │
   └─[对公转账，需审核]─→ pending_review ─[平台审核通过]─→ paid
                                          ─[驳回]─→ rejected

每条不可变，append-only。

### 10.4 agent_withdrawal.status

```
pending ─[代理财务审核]─→ processing ─[平台财务打款回写]─→ paid
   │                          │
   │                          └─[平台驳回]─→ rejected
   │
   └─[代理管理员取消]─→ cancelled
```

pending ─[平台财务对账]─→ confirmed ─[平台批次发放]─→ paid

   └─[退款触发]─→ reverted

### 10.6 agent_commission_distribution（代理给销售的）

```
pending ─[代理财务对账]─→ confirmed ─[代理财务打款]─→ paid

   └─[退款触发]─→ reverted
```

## 十四、测试要点 / 端到端验证

### 14.1 必测路径

1. **代理首次登录 4 步引导**：登录 → 上传 logo → 设价 → 进货 → 邀请销售 → 进入工作台
2. **代理体系销售卖货**：销售生成链接 → 客户落地代理镜像站 → 注册 → 付款 → 平台开通 → 代理余额扣减 → 平台 + 代理 + 销售三方佣金正确生成
3. **余额不足拦截**：余额为 0 时客户付款 → 订单 blocked_insufficient_balance → 通知代理 + 客户经理 → 代理补货 → 自动重试开通
4. **进货流程**：对公进货 → 上传凭证 → 平台审核 → 到账 → 余额 + 日志
5. **提现流程**：代理申请 → 代理财务审核 → 平台财务打款 → 回写流水 → 余额扣减
6. **退款冲销**：客户退款 → 代理余额回退 + 销售佣金 reverted + 代理佣金 reverted
7. **品牌定制**：改 logo → 实时预览 → 保存 → CDN 生效 → 客户端可见
8. **价格自定义**：设售卖价 < 进货价 → 拒绝；售卖价 < 基准价 × 50% → 警告
9. **跨代理数据隔离**：代理 A 用 cookie 调代理 B 的接口 → 401 + 安全告警
10. **代理冻结**：平台冻结代理 → 镜像站 403 + 名下销售 / 财务同步冻结 + 客户继续可用
11. **三角色权限边界**：代理销售调"招销售"接口 → 401；代理财务调"改品牌"接口 → 401
12. **USCC 跨代理去重**：代理 A 销售录入 USCC X，代理 B 销售再录入同 X → 拒绝
13. **客户 agent_id 不可改**：尝试修改 tenant.agent_id → 数据库触发器拒绝

### 14.2 边界测试

- 余额恰好等于客户付款 → 扣减成功，余额 = 0
- 余额比客户付款少 1 元 → 扣减失败，订单 blocked
- 多客户并发付款 → 余额扣减串行化（行锁）
- 同代理两个销售同时录入同 USCC → 先到先得
- 进货金额 1 分钱（最小值边界）→ 拒绝
- 代理被冻结的同时销售正在用 → 销售下一次请求被踢出
- 自定义域名证书过期 → 镜像站 fallback 到子域名

## 十五、风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| **跨代理数据越权** | 严重信任 / 法律事故 | 后端中间件强制 + 单测 + 月度渗透测试 + 安全告警 |
| **代理余额对账失败** | 财务损失 | append-only 日志 + 每日对账任务巡检 + 异常告警 |
| **代理用低价倾销 / 串货** | 品牌损害 / 其他代理投诉 | 5/20 不做防串货，靠合同条款 + 月度抽查；V2 上区域机制 |
| **代理私自截留客户付款** | 资金风险 | 客户付款直接进平台账户，代理只能提现；非代理收款 |
| **代理被攻击：销售账号被盗** | 销售名下客户被乱改 | 强 MFA + 异常登录告警 + 销售操作审计 |
| **客户感知到这是镜像站不是代理自家** | 信任流失 | 严格品牌定制 + 文档措辞中性 + 客服话术统一 |
| **进货金额错算 / 漏算** | 财务纠纷 | 进货 → 余额变化 → 日志 全程审计 + 月度代理对账单 |
| **代理拿 30% 佣金但内部给销售比例不透明** | 销售投诉代理 | 代理在 M3 销售档案显示佣金率明细 + 销售自己可在 M11 看 |
| **代理批量套利**（开虚假客户骗佣金） | 财务损失 | 客户必须 USCC + 实名认证 + 反欺诈风控 |
| **5/20 上线后改 V2 破坏代理数据** | 信任崩塌 | agent_id 准确挂钩每笔订单 + 数据迁移测试 + 灰度上线 |
