## 0. 产品全景（重新校准）

本次会议之后，代理产品矩阵正式拆成 **两条线**：

| 产品线 | 定位 | 用户 | 本次优先级 |
|---|---|---|---|
| **代理商 Web 管理平台**（本次会议新增的核心） | 我司自有域名/品牌，扒火山设计语言；内含：客户侧（充值、咨询、资讯、应用入口）+ 后台（销售、交付、官方管理员） | 终端企业、销售、交付、我司官方管理员 | **P0（立即启动）** |
| **ArkClaw 桌面客户端**（之前讨论的 Electron 套壳 + 本地桥） | 套壳 + 本地能力，Coding/办公场景 | 终端企业员工 | **P1（本月不启动，Web 平台稳定后再排）** |

关键原则（会议明确）：
- **账号不过我们**，但**租户管理过我们**——火山给 API，我们做和火山体验一致但是自有域名/品牌的"多租户管理平台"。
- **UI 可以直接扒火山**，融合"云脑智联 × 火山引擎"双 logo 定位，文档类链接直跳火山官网。
- 分销**只做一级**，不做多级代理，避免踩传销红线。

## 2. 待决策的业务规则（需老板/商务拍板）

以下问题不决策无法进入详设，建议 2-3 天内碰一次：

| # | 问题 | 选项 | 默认建议 |
|---|---|---|---|
| D1 | **佣金分成周期** | 一年 / 十个月 / 客户生命周期 | 先按"客户首次付款后 12 个月"执行，12 个月后可续签 |
| D2 | **佣金阶梯规则** | 固定比例 / 按金额阶梯 / 按产品阶梯 | 阶梯式：<10 万 5%，10-50 万 8%，>50 万 10%（待确认）|
| D3 | **客户去重维度** | 名称 / 统一社会信用代码 / 联系方式 / 多字段组合 | **以统一社会信用代码为主键**（企业唯一）+ 联系方式辅助告警 |
| D4 | **充值链路** | 客户直充火山 / 我们代收后同步扣火山额度 | **代收款**（才能算分成、做媒体资讯/咨询等增值）|
| D5 | **支付渠道** | 微信 / 支付宝 / 对公 / 全要 | 三个全要，对公必须（企业客户主流）|
| D6 | **咨询智能体接入** | 自建客服智能体 / ArkClaw 内置 | 用"云脑智联自己的智能体"（会议明确）|
| D7 | **分销注册链路** | 分享链接 / 邀请码 / 两者都有 | 分享链接（带销售 ID 参数），注册时绑定 |
| D8 | **客户经理分配** | 手动 / 按行业/地域自动 | 首版手动；留自动分配扩展点 |
| D9 | **媒体资讯**定位 | 客户发自己的 / 我们发行业内容 / 两者都有 | 两者都要，分"官方资讯"和"企业资讯"两栏 |

## 3. 产品设计任务（启动条件：API 清单到位 + 业务规则决策完成）

### 3.1 信息架构与权限矩阵

- [ ] 画出**三层角色**的完整权限矩阵（Excel / Notion 表）：
  - **我司官方管理员**：全平台可见，所有企业 + 所有销售 + 所有订单；
  - **我司交付 / 运维**：绑定配置、网络配置、企业开通后的技术支持；
  - **销售**：仅自己名下客户 + 佣金明细；
  - **客户企业管理员**：只看自己企业；能看到技能中心、应用中心、知识中心、用户管理等；
  - **客户企业普通员工**：只用，不管理。
- [ ] 权限粒度明确到按钮级，避免"看到但点不了"的二义性。

### 3.2 原型稿清单（按页面划分）

**客户侧（登录后）**
- [ ] 首页仪表盘（余额、用量、最近活动）
- [ ] 自助充值（月/季/年 + 微信/支付宝/对公 + 充值历史 + 消费明细）
- [ ] 应用入口（嵌入 / 跳转 ArkClaw 能力，50+ 技能）
- [ ] 企业套餐咨询（需求表单 + 客户经理信息 + 预约演示）
- [ ] 媒体资讯（图文编辑 + 分类 + 发布/下架 + 前端展示）
- [ ] 技能中心 / 应用中心 / 知识中心 / 用户管理（扒火山，做成本企业隔离视图）
- [ ] 飞书/企微/钉钉/微信绑定入口（如果火山 API 支持直传）

**我司后台 · 官方管理员**
- [ ] 企业列表（所有下游企业 + 基本信息 + 用量 + 状态）
- [ ] 企业详情钻取（技能、应用、知识、成员、订单）
- [ ] 全局统计（活跃度、收入、消耗、续费漏斗）
- [ ] 销售管理（销售账号 + 客户绑定关系 + 佣金规则配置）

- [ ] 绑定管理（飞书/企微/钉钉/微信的初始配置流程）
- [ ] 网络配置（内网打通、虚拟专网等，仅运维可见）
- [ ] 工单 / 客户问题跟踪

**我司后台 · 销售**
- [ ] 我的客户池（增删改查 + 去重校验提示）
- [ ] 销售仪表盘（客户数、本月新客、本月回款、预估佣金）

- [ ] 分享链接生成 + 扫码注册落地页

- [ ] 客户去重校验：新增客户时实时查询统一社会信用代码，如重复，明示"已被销售 X 录入"（不泄露销售身份可改显"已被他人录入"）；

## 4. 技术前置任务（与产品设计并行）

### 4.1 架构选型
- [ ] 前端：Vue 3 / React（与团队现有栈一致，优先复用）；UI 库对齐火山设计（Arco Design 是火山出品 🎯 **强烈推荐**）；
- [ ] 后端：Go / Java / Node（按团队选）；
- [ ] 数据库：MySQL / PostgreSQL；
- [ ] 部署：容器化；Nginx + 负载均衡；独立域名 + HTTPS。

### 4.2 核心数据模型（初稿）

```
tenant          租户（= 下游企业客户）
user            用户（跨租户，通过 tenant_user 关联）
tenant_user     租户-用户-角色映射
role            角色（官方管理员 / 运维 / 销售 / 企业管理员 / 企业员工）
permission      权限点
sales           销售档案
sales_customer  销售-客户绑定（含绑定时间、来源 = 分享链接/手动）
order           订单（充值 / 套餐）
commission      佣金流水（订单触发，按规则计算）
commission_rule 佣金规则配置（阶梯、周期）
arkclaw_mapping 本系统 tenant ↔ 火山 ArkClaw 租户 ID 映射
binding         第三方绑定（飞书/企微/钉钉/微信 × 租户）
news            媒体资讯（分官方/企业）
consult_lead    企业套餐咨询留资
```

### 4.3 火山 API 接入层
- [ ] 独立封装 `volc-arkclaw-sdk`（内部库），所有火山调用走此层；
- [ ] 统一处理鉴权、重试、熔断、日志；
- [ ] 本地 tenant ID 与火山 tenant ID 的双向映射表。

### 4.4 支付接入
- [ ] 微信 / 支付宝 开放平台商户资质准备；
- [ ] 对公转账：后台上传凭证 + 财务审核工单流程；
- [ ] 支付回调 → 订单状态 → 触发火山额度同步 + 佣金计算。

### 4.5 分享链接 / 归因
- [ ] 短链服务（自建 or 腾讯云短链）+ 销售 ID 加密参数；
- [ ] 落地页先埋 Cookie / localStorage，注册成功即回写绑定关系；
- [ ] 防刷：同 IP 限制 + 已注册用户不重绑。

### 4.6 权限 / 多租户隔离
- [ ] 数据库行级隔离（`tenant_id` 过滤）+ 应用层 RBAC 中间件；
- [ ] 审计日志（谁在何时对哪个租户做了什么）。

- 所以**先做 Web、晚做桌面**，是省事不是重做。
