# ArkClaw 功能清单

本目录用于维护 ArkClaw 平台自己的功能清单。参考 `0-assets/功能清单参照私有云V5.3-电子签章功能清单 (2025年6月6日—9月5日).xlsx` 的分层结构，但不直接沿用 Excel 作为唯一源文件。

## 文件分工

| 文件 | 用途 |
|---|---|
| `arkclaw-demo-features.csv` | Demo 待实现功能主数据源。按可验收动作拆分，用于产品排期和验收 |
| `arkclaw-dev-implemented-features.csv` | 开发已实现功能主数据源。记录 `/dev` 中已有真实实现证据的能力，部分实现也纳入 |
| `arkclaw-features.xlsx` | 多 Sheet 评审版，包含 Demo 主表和按角色分表 |
| `by-role/*.csv` | 按 `角色域` 从 Demo 待实现功能拆分出来的角色清单 |
| `arkclaw-features.csv` | 旧版粗粒度清单，保留为历史来源，不再作为主维护文件 |
| `arkclaw-features.md` | 阅读版摘要。用于评审时快速理解模块边界和优先级 |
| `exports/` | 备用导出目录 |

维护规则：**只在两份新 CSV 中维护明细，按角色 CSV 与 XLSX 视为评审导出产物**。

编码规则：CSV 使用 **UTF-8 with BOM**，方便 Excel / WPS 直接打开时正确识别中文。

## 字段说明

| 字段 | 说明 |
|---|---|
| `feature_id` | 功能项唯一编号，按域前缀编号，例如 `TEN-001`、`ADM-001` |
| `角色域` | 功能所属业务域，例如客户侧、销售侧、官方管理员后台 |
| `一级功能` | 顶层模块 |
| `二级功能` | 子模块或页面 |
| `三级功能` | 具体能力分组 |
| `四级功能` | 更细的动作、状态或规则 |
| `功能点` | 可单独开发、测试和验收的动作 |
| `功能说明` | 功能项说明，强调业务能力而不是 UI 细节 |
| `优先级` | `T0`、`P1`、`P2`、`P3` |
| `实现说明` | 仅用于开发已实现表，说明 `/dev` 中对应的实现证据 |
| `验收要点` | 最小验收口径 |
| `备注` | 范围边界、后置原因或实现注意点 |
| `是否有原型` | 有原型 Demo 时填完整地址，例如 `http://localhost:5160/register`；无则填 `/` |
| `开发是否已实现` | `是` / `否`，Demo 表按人工语义映射标记；本轮口径为有实现证据即算 `是` |
| `版本` | 已实现且有原型为 `v1.0`，未实现但有原型为 `v1.1`，无原型统一为 `v2.0` |

## 优先级口径

| 优先级 | 含义 |
|---|---|
| `T0` | 主流程闭环必需，缺失会阻断 Beta1 验收 |
| `P1` | 不阻断主链路，但明显影响运营效率、异常处理或客户体验 |
| `P2` | 规模化、精细化、管理增强或技术运维增强 |
| `P3` | 高风险、低确定性或短期不建议开放的能力 |

## 粒度口径

| 规则 | 含义 |
|---|---|
| 可验收动作 | 一行只表达一个可以被开发、测试、验收的功能点 |
| 编号粒度 | 细分功能继续使用后缀编号，例如 `TEN-014-01`、`TEN-014-02` |
| 已实现口径 | 只要 `/dev` 中已有真实实现证据即可进入开发已实现表；半闭环能力在 `备注` 写明缺口 |
| 实现映射 | Demo 表的 `开发是否已实现` 采用人工语义映射，不按父功能整组命中 |
| 规划口径 | Demo 待实现表只表达设计目标，不表达当前实现状态 |
| 原型口径 | `是否有原型` 使用 `http://localhost:5160` 下可访问的 Demo 页面地址；无可访问页面填 `/` |

## 维护流程

1. 新增 PRD 或评审结论后，先更新 `arkclaw-demo-features.csv`。
2. `/dev` 技术实现发生变化后，按前后端闭环更新 `arkclaw-dev-implemented-features.csv`。
3. 如果功能被砍掉，不删除行，在 `备注` 中说明范围变化或暂缓原因。
4. 按角色评审时使用 `by-role/*.csv` 或 `arkclaw-features.xlsx` 中对应角色 Sheet。
5. 对外评审前，重新生成 `arkclaw-features.xlsx`。
