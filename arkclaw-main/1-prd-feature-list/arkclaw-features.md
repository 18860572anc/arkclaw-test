# ArkClaw 平台功能清单阅读版

主数据源见 `arkclaw-demo-features.csv` 和 `arkclaw-dev-implemented-features.csv`。本文档只做评审阅读和范围说明。

## 版本口径

| 版本 | 范围说明 |
|---|---|
| Beta1 | 跑通销售获客、客户注册、对公购买、后台审核、开通任务、运维交付、席位分配、成员可用、佣金可见 |
| Beta1.1 | 补齐运营效率、异常处理、财务协作、通知和审计等增强能力 |
| Beta2 | 规模化管理、细粒度权限、监控告警、自动化运维、复杂结算等能力；无原型功能统一标为 `v2.0` |
| 暂缓 | 高风险或短期不建议开放的能力 |

## 功能域总览

| 角色域 | 核心范围 | 当前重点 |
|---|---|---|
| 公共与系统 | 登录、企业开户注册、销售归属、角色路由、通知、审计 | 先支撑 Beta1 主链路登录、企业开户注册和角色路由；独立手机号注册与按钮级权限移出当前清单 |
| 客户侧 | 企业空间、购买账单、席位、Claw、能力中心、组织、安全 | Beta1 重点是注册、购买、对公、席位和成员工作台；内容资讯已移出当前清单 |
| 销售侧 | 客户池、分享链接、开通跟进、佣金明细 | Beta1 必须支撑客户归属和佣金可见；销售仪表盘移出当前清单 |
| 官方管理员后台 | 企业、订单、对公审核、开通任务、销售、佣金、套餐、员工 | Beta1 是主流程中台；内容运营、字典管理、监控告警和高危操作已移出当前清单 |
| 交付运维后台 | 工单队列、支持工单、客户上下文、订单与开通 | Beta1 收敛为有原型的运维流转；无原型项和健康巡检已移出当前清单 |
| 代理镜像站 | 代理客户、订单、销售、财务、网站、券、员工 | 用于代理自运营闭环 |
| 财务侧 | 对公、发票、佣金、提现、结算 | Beta1 后可逐步从官方后台拆出专属视角 |
| API 与开放能力 | API Key、调用记录、服务目录、渠道状态、计费 | 当前以客户侧 API 密钥台为主 |

## 当前表格产物

| 文件 | 内容 |
|---|---|
| `arkclaw-demo-features.csv` | Demo 待实现功能，按可验收动作拆分 |
| `arkclaw-dev-implemented-features.csv` | `/dev` 开发已实现功能，仅记录前后端闭环 |
| `arkclaw-features.xlsx` | 多 Sheet 评审版，包含主表、开发已实现表和按角色分表 |
| `arkclaw-features.csv` | 旧版粗粒度来源表，保留用于追溯 |

## Beta1 主链路功能

| 环节 | 关键功能项 |
|---|---|
| 获客与归属 | 销售分享链接、客户池、新增客户、咨询留资分配 |
| 注册与开户注册 | 手机号注册、企业信息、USCC 去重、销售归属绑定 |
| 购买与对公 | 标准套餐下单、支付订单、付款凭证、对公审核、驳回补充 |
| 开通交付 | 订单 paid 后生成开通任务、工单派发、运维处理、异常回填 |
| 席位可用 | 企业级激活、席位池、成员邀请、席位分配、成员进入工作台 |
| 佣金可见 | 佣金规则、佣金流水、销售侧 pending 状态展示 |

## 后置原则

Beta1 不做完整运维平台、复杂佣金结算、销售排行榜、系统监控告警、角色权限矩阵、自动银行流水识别、真实支付通道、真实火山资源映射控制台。相关能力在 Demo 待实现清单中保留，并通过优先级和备注说明边界。

## 与参考 Excel 的差异

参考 Excel 采用多工作表和合并单元格组织，适合评审展示。ArkClaw 清单采用两份 CSV 做主数据源，并导出多 Sheet XLSX：一张记录 Demo 待实现功能，一张记录开发已实现功能，其余按角色域拆分。新版表格不再保留旧版 `状态`、`页面路由`、`来源PRD` 字段；原型入口统一沉淀到 `是否有原型` 列，填 `http://localhost:5160` 下可访问的 Demo 地址，无则填 `/`。
