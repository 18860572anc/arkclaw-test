## 💡 建议修复方案

### 方案一：修复 Token 传递
```typescript
// httpClient.ts - 确保 Token 正确携带
const httpClient = new HttpClient({
baseURL: baseUrl,
headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
}
});
```

### 方案二：适配镜像站登录接口
```typescript
// authApi.ts - 根据环境选择登录方式
if (env === 'mirror') {
// 镜像站可能使用 GET 方法
const response = await httpClient.get('/api/auth/login', {
    params: { username, password, domain }
});
} else {
const response = await httpClient.post('/api/auth/login', { username, password, domain });
}
```

### 方案三：优化异常测试设计
```typescript
// workflow.test.ts - 独立每个测试的登录状态
test('错误密码登录应报错', async () => {
// 直接测试，不依赖 beforeAll
try {
    await authApi.login('invalid', 'wrong');
} catch (error) {
    expect([400, 401]).toContain(error.response?.status);
}
});
```
