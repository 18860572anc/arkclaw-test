## 🎯 快速上手（3步）

### 第1步：阅读主测试文档
打开 `ArkClaw_Beta1_功能测试覆盖文档.md`，重点阅读：
- **第0-1章**：了解测试策略和范围（5分钟）
- **第2章**：E2E主流程测试（核心场景）
- **第8章**：验收标准和发布准入条件

### 第2步：准备测试环境
1. 确保前端Mock服务运行在 `http://localhost:5160`
2. 使用第7章的预设测试账号登录不同角色
3. 确认Mock数据正常加载

### 第3步：开始执行测试
1. 复制 `测试用例执行记录模板.csv` 为新的Excel文件
2. 按以下顺序执行测试：
   ```
   E2E主流程 → 按角色分域 → 权限隔离 → UI规范 → 异常流程
   ```
3. 记录每个用例的执行结果（通过/失败/阻塞）
4. 发现缺陷时记录缺陷ID和截图路径

## 🔍 常见问题

**Q: 如何判断测试是否通过？**
A: 对照每个用例的"验收标准"部分，所有checkbox都打勾才算通过。

**Q: 发现缺陷怎么办？**
A: 
1. 在测试记录表中标记"失败"
2. 记录缺陷ID（如JIRA ticket号）
3. 截图保存证据
4. 描述复现步骤

**Q: Mock数据如何配置？**
A: 参考主文档第7.2节的Mock数据模板，修改 `frontend/src/services/*MockApi.ts` 文件。

**Q: UI不符合规范怎么处理？**
A: 对照 `PRD参考文档/FRONTEND_GUIDELINES.md` 的具体章节，记录不符合项并反馈给前端开发。
