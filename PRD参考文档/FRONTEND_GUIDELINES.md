# ArkClaw Frontend Guidelines

本文档是 ArkClaw 前端唯一视觉规范入口。后续实现必须优先遵循这里的规则，再参考 `0-assets/ArkClaw` 原型和 `frontend/src/styles.css` 已落地样式。除非产品原型明确变更，不允许临时发明新的颜色、字号、布局模式或组件风格。

## 1. 设计原则

ArkClaw 是企业控制台产品，界面目标是高效、清晰、可扫描，而不是营销展示。

- 信息密度要偏高，但视觉要克制。优先用表格、列表、轻量卡片、抽屉和表单承载工作流。
- 页面必须服务真实操作，不做 landing page、hero、装饰性大图、装饰性渐变背景。
- 原型优先。实现视觉和布局时，以 `0-assets/ArkClaw` 下同名或同模块原型为准。
- Arco Design React 是默认 UI 库。自定义 CSS 只用于布局、视觉 token、密度调整和原型对齐。
- 不允许嵌套卡片。页面区域用白底面板或自然布局；卡片只用于重复对象、统计块、选择项和真正需要边界的工具区。
- 禁止新增大面积紫色、蓝紫渐变、装饰性背景、营销式 hero、纯氛围视觉。

## 2. 技术栈和 UI 库

- Framework: React + TypeScript + Vite。
- UI Library: `@arco-design/web-react`。
- Routing: `react-router-dom`。
- Charts: `recharts`。
- 样式入口：`frontend/src/styles.css`。
- 路由布局元数据入口：`frontend/src/config/appRoutes.tsx`。

实现规则：

- 优先使用 Arco 组件：`Button`、`Input.Search`、`Input`、`Select`、`Tabs`、`Drawer`、`Table`、`Form`、`Checkbox`、`Radio`、`Avatar`、`Empty`。
- 图标优先使用 `@arco-design/web-react/icon`，不要手写 SVG。
- 搜索框必须使用 `Input.Search`。
- 表单输入控件必须使用 Arco 组件并通过统一 CSS 覆盖视觉，不要单独写原生 input。
- 不引入第二套 UI 组件库，除非原型明确要求且现有 Arco 组件无法完成。

## 3. Typography

字体栈固定：

```css
font-family: Inter, "PingFang SC", "Microsoft YaHei", system-ui, sans-serif;
```

字号和用途：

| 用途 | 字号 | 字重 | 行高 |
| --- | ---: | ---: | ---: |
| 辅助说明、表格补充信息、占位说明 | 12px | 400 | 18-20px |
| 控制台正文、表格单元格、表单标签 | 13px | 400/500 | 18-22px |
| 区块标题、卡片标题、表单小标题 | 14px | 600/700 | 20-22px |
| 页面标题、二级页标题 | 16px | 500/600/700 | 24-28px |
| 一级概览标题 | 18px | 700 | 28px |
| 数字指标 | 26px | 700 | 30px |

文字规则：

- 字距固定为 `0`，不使用负字距。
- 页面标题不做超大字号。
- 控制台内容以 12/13/14px 为主。
- 表格、抽屉、表单内标题不得使用 hero 级字号。
- 长文本必须省略或换行，不允许溢出容器。

## 5. Spacing Scale

间距基准为 4px。允许使用以下刻度：

```text
4 / 6 / 8 / 10 / 12 / 14 / 16 / 18 / 20 / 24 / 28 / 32 / 38 / 48 / 56
```

常用规则：

- 顶栏高度：`56px`。
- 左侧导航宽度：`200px`。
- 一级页面内容 padding：`20px 76px 38px 28px`。
- 二级页面内容 padding：`20px 28px 38px`。
- 页面区块垂直间距：`24px` 或 `48px`。
- 表单项间距：`18px`。
- 卡片内部 padding：`14px`、`16px` 或 `18px`。
- 工具条 gap：`12px` 或 `16px`。
- 抽屉字段间距：`18px`。

不得使用随机间距值。若原型中有特殊偏移，优先抽象成页面级规则，而不是散落在组件里。

## 6. Layout Rules

### 6.1 App Shell

- 顶部导航固定高度 `56px`，白底 `#FFFFFF`，底边 `#E5E8EF`。
- 左侧导航宽度固定 `200px`，背景 `#F2F5FA`。
- 工作区高度为 `calc(100vh - 56px)`，内部滚动。
- 右侧浮动帮助按钮固定在视口右侧中部，尺寸 `38px`。

### 6.2 一级页面

一级页面包括概览、列表、中心页、表格页、配置列表页。

- 保留顶部导航和左侧导航。
- 内容区偏左展开，面向浏览和批量操作。
- 可以使用 breadcrumb，除非原型不展示。
- 页面顶部通常为标题 + 主操作按钮。
- 工具条位于 tabs 或标题下方，搜索框靠左，刷新/次操作靠右。

示例：

- `/tenant/templates`
- `/tenant/overview`
- `/tenant/claws`
- `/tenant/seats`
- 安全、网络、运维、用户管理等模块一级页。

### 6.3 专注二级页面

二级页面包括创建、编辑、配置、独立详情编辑页。

- 隐藏左侧导航。
- 保留顶部导航。
- 隐藏 breadcrumb。
- 内容区域居中，最大宽度按页面类型控制。
- 页面内部必须提供返回入口。
- 页面底部必须有固定操作栏。
- 底部操作栏从视口左侧开始，占满宽度，不给左侧栏预留空位。

当前已落地的二级页：

- `/tenant/templates/create`
- `/tenant/overview/update/model`

新增二级页必须在 `frontend/src/config/appRoutes.tsx` 中设置：

```ts
layoutMode: 'detail',
showBreadcrumb: false,
```

### 6.4 抽屉和弹窗

- 抽屉保留在一级页上下文中，不升级为二级路由。
- 抽屉打开时左侧导航仍保留在背后。
- 模板批量派发抽屉宽度锁定为约 `748px`。
- 抽屉 body padding 使用 `14px 16px 0`。
- 抽屉 footer padding 使用 `10px 16px 12px`。

## 7. Component Rules

### 7.1 Buttons

- 主操作使用 Arco `Button type="primary"`，主色 `#165DFF`。
- 次操作使用默认按钮。
- 图标型操作使用图标按钮，不写“刷新”“删除”这类可由图标表达的文字，必要时加 `title` 或 `aria-label`。
- 深色按钮只用于原型明确需要的强调动作，背景 `#1D2129`，hover `#272E3B`。
- 小型控制台按钮高度以 Arco small/default 密度为准，不手写大按钮。

### 7.2 Search

搜索框必须使用 `Input.Search`。

固定视觉：

- 背景：`#FFFFFF`。
- 默认边框：`#DCE3EE`。
- hover 边框：`#C8D4E3`。
- focus 边框：`#94B5FF`。
- focus 阴影：`0 0 0 2px rgba(22, 93, 255, .10)`。
- 圆角：`4px`。
- placeholder：`#B4BFCC`。
- 图标：`#86909C`，hover `#4E5969`。
- 搜索框必须始终可见边框，不能做成无边框白底，也不能依赖父容器边框伪装。

禁止使用灰底搜索框。

### 7.3 Input / Select / TextArea

所有输入框、选择器、文本域默认白底。

- 背景：`#FFFFFF`。
- 默认边框：`1px solid #DCE3EE` 或 `1px solid #E5E6EB`。
- hover 边框：`#C8D4E3`。
- 聚焦边框：`#94B5FF`。
- 聚焦阴影：`0 0 0 2px rgba(22, 93, 255, .10)`。
- 圆角：`4px` 或 `6px`。
- Select 必须显示完整外边框，不能做成文字下拉或无描边白底。
- 二级 detail 页中的 `Input`、`Select`、`TextArea` 必须和一级页保持同一套边框规则，不能因为 detail 布局切换成“只有白底、没有边框”的状态。
- 如果 Arco 组件在不同页面渲染出的 wrapper 结构不一致，页面级必须补充统一控件类，例如 `detail-control`，直接把边框画在实际承载视觉的节点上。

本条需要补充一个明确的实现约束：

- 不要假设 Arco 输入类控件的外框一定在同一个 DOM 层级。实际项目中：
  - `Input` 可能直接渲染为 `.arco-input`
  - 带前后缀、清空、组合态的 `Input` 可能走 `.arco-input-inner-wrapper`
  - `TextArea` 可能是 `.arco-textarea`
  - `TextArea` 外层还可能带 `.arco-textarea-wrapper`
- 页面做视觉覆盖时，必须先确认“真实承载边框”的节点，再写样式；不能只凭经验去改 `.arco-input-wrapper` 或其他猜测类名。
- 如果页面上出现“输入框白底但看不到边框”的情况，优先排查是否样式打错到了错误节点，而不是先怀疑 Arco 底层库异常。
- 结论：对 Arco 输入类控件做页面级视觉统一时，至少检查并按需覆盖 `.arco-input`、`.arco-input-inner-wrapper`、`.arco-textarea`、`.arco-textarea-wrapper` 这几类真实节点。

二级页中必须用 `.app-shell--detail` 统一覆盖输入类控件为白底加边框，避免 Arco 默认灰底或边框缺失泄漏。

### 7.4 Forms

- 表单 label 使用 13px。
- 必填星号使用 `#F53F3F`。
- 表单项间距 `18px`。
- 二级页表单最大宽度优先 `600px`。
- 创建/编辑页使用蓝色左竖线区块标题：
  - 竖线宽度 `3px`
  - 颜色 `#165DFF`
  - 标题字号 `14px`

### 7.5 Tabs

- Tabs 用于一级页面内容切换或抽屉内部来源切换。
- Tabs 下方不得额外重复标题。
- 原型中只需要 tab header 时，隐藏 tabs content。
- 激活态使用 `#165DFF`。

### 7.6 Cards

- 卡片圆角最多 `8px`。
- 控制台面板优先 `4px` 或 `5px`。
- 卡片背景 `#FFFFFF`，边框 `#E5E8EF` 或 `#EDF0F5`。
- 卡片阴影只允许轻量：`0 4px 10px rgba(29, 33, 41, .03)`。
- hover 只改变边框、轻阴影，不做夸张位移。
- 不允许卡片套卡片。

### 7.7 Tables and Lists

- 表格用于高密度数据。
- 表格工具条在表格上方，左侧为搜索/筛选，右侧为刷新/批量操作。
- 控制台表格默认不使用竖向分隔线。列与列之间优先依靠留白、列宽和横向分隔建立节奏，不开启类似 `borderCell` 的整列边框表现，除非原型明确要求。
- 行内操作优先使用图标或简短文字按钮。
- 文本过长必须省略，不撑开列宽。

### 7.8 Drawer

- 抽屉标题字号遵循 Arco 默认控制台密度。
- 字段 label 使用 13px。
- 抽屉内部字段垂直 gap `18px`。
- 双栏选择器使用等宽两栏，中间 `1px` 分割线。
- 空态居中，文字 `12px`，颜色 `#86909C`。

### 7.9 Empty State

- 空态居中。
- 图标使用低饱和灰：`#ADB4C0`。
- 文案 12px，颜色 `#86909C`。
- 空态不放大标题，不加营销说明。

### 7.10 Fixed Footer

二级页底部操作栏固定在视口底部。

- 背景：`rgba(255, 255, 255, .96)` 或 `rgba(255, 255, 255, .98)`。
- 顶边：`#E5E6EB` 或 `#EDF0F5`。
- 高度：模型配置类 `48px`；创建模板类使用 `10px 16px` padding。
- detail 布局中 `left: 0`。
- 一级带左栏页面若有固定底栏，才可使用 `left: var(--side-nav-width)`。

## 8. Page Patterns

### 8.1 Template Center

- 一级页，保留左栏。
- 标题 `Claw 模板中心`，右侧主按钮 `批量派发模板`。
- Tabs：`模板市场`、`自定义模板`。
- 搜索框宽度约 `210px`。
- 模板卡片宽度 `210px`，圆角 `8px`。
- 自定义模板无数据时使用居中空态。

### 8.2 Template Create

- 专注二级页，隐藏左栏。
- 内容居中，表单最大宽度 `600px`。
- 顶部为返回图标 + `创建模板`。
- 基本信息和高级配置使用蓝色左竖线标题。
- 输入控件必须白底。
- 页面底部固定取消/确定操作栏。

### 8.3 Model Config Edit

- 专注二级页，隐藏左栏。
- 内容居中，主编辑区域宽度 `600px`。
- 顶部为返回图标 + `模型配置`。
- 提示条背景 `#EEF4FF`。
- 底部固定取消/确认操作栏，全宽。

### 8.4 Dispatch Drawer

- 属于模板中心一级页上下文。
- 不隐藏左栏。
- 抽屉宽度约 `748px`。
- 模板来源用轻量 segmented switch。
- 员工选择器使用左右双栏，左栏全部员工，右栏已选员工。

## 9. Responsive Rules

### 1320px 以下

- 顶部右侧弱导航项可隐藏。
- 二级页仍保持居中，但可减少左右空白。
- 不改变核心字号。

### 960px 以下

- 左侧导航隐藏。
- `.workspace__inner` padding 改为 `16px`。
- 顶部中间搜索和右侧 actions 隐藏。
- 多列栅格改为单列。
- 工具条、标题行、表单辅助区域允许纵向排列。
- 搜索框、卡片、表单宽度变为 `100%`。
- 固定底栏 `left: 0`。

禁止使用 viewport width 动态缩放字体。

## 11. Implementation Checklist

新增或修改页面前必须检查：

- 是否有对应原型。
- 页面是一级页还是专注二级页。
- 是否需要隐藏左侧栏。
- 是否需要底部固定操作栏。
- 输入框、搜索框、选择器、文本域是否都是白底。
- 是否使用了规定色值和 spacing scale。
- 是否避免了嵌套卡片、营销 hero、大面积渐变。
- 是否在 960px 以下不溢出、不重叠、不出现固定宽度撑破。

如果实现和本文档冲突，先改实现；如果原型更新导致本文档过期，先更新本文档，再改实现。
